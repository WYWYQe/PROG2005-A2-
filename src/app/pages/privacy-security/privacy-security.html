Privacy & Security Analysis

This page explains important privacy and security considerations for the inventory management system.

1. Sensitive Data Should Not Be Stored in the Frontend

This application stores inventory information such as item name, price, quantity, supplier name, and stock status. It should not store highly sensitive information such as passwords, payment details, secret keys, or personal identity data directly in the browser.

2. Input Validation Is Important

All user input should be validated before it is saved. For example, quantity and price should only accept valid numbers, required fields must not be empty, and category or stock status should only allow predefined options.

This reduces incorrect data entry and helps protect the system from unexpected or unsafe input.

3. Local Storage Has Risks

If localStorage is used to keep data after refreshing the browser, the data remains on the user’s device. This is convenient, but it also means other users on the same device may access the stored information.

For real business systems, sensitive data should be protected on a secure backend instead of relying only on browser storage.

4. Avoid Cross-Site Scripting (XSS)

User-entered text such as notes or supplier names should be displayed safely. Developers should avoid inserting unsafe HTML directly into the page unless the content is sanitized.

Angular helps reduce XSS risks because it escapes template output by default, which is safer than directly writing raw HTML into the page.

5. Principle of Least Privilege

In a larger real-world system, users should only have access to the functions they need. For example, some users may only be allowed to view stock, while managers can add, edit, or delete inventory records.

6. Error Messages Should Be Clear but Safe

Error messages should help users understand what went wrong, such as “Item name is required” or “Price must be a valid number”. However, the system should avoid exposing technical internal details that could create security risks.

7. Summary

This inventory system is suitable for learning frontend development and Angular application structure. To improve privacy and security in a real business environment, the application should use stronger validation, secure authentication, role-based access control, and backend data protection.
